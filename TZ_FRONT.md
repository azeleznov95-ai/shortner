# Техническое задание (ТЗ)
## Frontend для сервиса сокращения ссылок

---

## 1. Общая информация

**Проект:** URL Shortener  
**Тип приложения:** Web (SPA или простое клиентское приложение)  
**Назначение:** Взаимодействие с backend-сервисом сокращения ссылок  
**Backend API:** REST (Spring Boot)  
**Аутентификация:** отсутствует  
**Роли пользователей:** не требуются

Frontend является вспомогательным интерфейсом для демонстрации работы backend API.

---

## 2. Цели frontend-части

Frontend должен:

- позволять пользователю создать короткую ссылку
- отображать результат (short URL)
- позволять перейти по короткой ссылке
- отображать информацию об ошибках API
- быть простым, понятным и минималистичным

SEO, сложный дизайн и анимации **не требуются**.

---

## 3. Технологические требования

Рекомендуемый стек (можно заменить аналогами):

- HTML5
- CSS3
- JavaScript (ES6+)
- Fetch API или Axios
- React / Vue / Vanilla JS — на усмотрение исполнителя

Обязательные требования:

- работа через HTTP-запросы к backend
- отсутствие серверного рендеринга
- сборка необязательна (можно без Vite / Webpack)

---

## 4. Структура интерфейса

### 4.1 Главная страница (`/`)

Элементы:

1. Заголовок приложения
    - «URL Shortener»

2. Форма создания короткой ссылки:
    - поле ввода оригинального URL
    - кнопка **«Сократить»**

3. Блок результата:
    - отображение short URL
    - кнопка «Скопировать» (опционально)
    - ссылка кликабельна

4. Блок ошибок:
    - отображение сообщения об ошибке (если API вернул ошибку)

---

## 5. Поведение интерфейса

### 5.1 Создание короткой ссылки

1. Пользователь вводит URL
2. Нажимает кнопку «Сократить»
3. Frontend отправляет POST-запрос в backend
4. При успехе:
    - отображается короткая ссылка
5. При ошибке:
    - отображается сообщение ошибки

---

## 6. Взаимодействие с API

### 6.1 Создание короткой ссылки

**Запрос:**

```
POST /shorten
Content-Type: application/json
```

**Тело запроса:**

```json
{
  "url": "https://example.com"
}
```

**Успешный ответ:**

```json
{
  "shortUrl": "http://localhost:8080/abc123"
}
```

---

### 6.2 Ошибки

Backend возвращает ошибки в формате:

```json
{
  "timestamp": "2026-01-01T12:00:00",
  "status": 400,
  "error": "Invalid URL",
  "path": "/shorten"
}
```

Frontend должен:

- отобразить `error`
- не ломать интерфейс
- очищать старые ошибки при новом запросе

---

## 7. UX требования

- минимальное количество элементов
- понятные подписи
- отсутствие сложных сценариев
- интерфейс должен работать без перезагрузки страницы

---

## 8. Ограничения

- аутентификация не требуется
- регистрация пользователей не требуется
- статистика переходов может не отображаться
- адаптивность — желательно, но не обязательно

---

## 9. Результат работы

Frontend считается готовым, если:

- пользователь может создать короткую ссылку
- корректно отображаются ошибки
- frontend стабильно работает с backend API
- код читаемый и структурированный

---

## 10. Примечание

Frontend не является основной частью проекта.  
Основной акцент проекта — backend и архитектура API.
